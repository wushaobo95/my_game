## Context

当前多重射击作为升级选项过于强力，导致后期DPS爆炸。通过伤害衰减虽然平衡了数值，但让多重射击变得"不爽快"。

更好的设计是：将多重射击改为可重复掉落的法宝，通过随机掉落来控制成长节奏。

## Goals / Non-Goals

**Goals:**
- 将多重射击从"每次升级都能选"改为"随机掉落"
- 保持多重射击的爽快感（无伤害衰减）
- 通过掉落概率控制成长节奏
- 最多+4额外投射物（防止无限叠加）

**Non-Goals:**
- 不改变多重射击的基本机制
- 不影响其他法宝和升级

## Decisions

### 决策1：可重复掉落的法宝
- **选择**: 新增法宝 id: 113，可重复掉落最多4次
- **原因**: 
  - 法宝掉落是随机的，不是每次升级都能获得
  - Boss掉落法宝，间隔45秒，成长节奏可控
  - 最多4次限制防止无限叠加
- **备选方案**: 
  - 保持伤害衰减 → 不爽快
  - 作为升级但限制次数 → 实现复杂

### 决策2：修改 trySpawnItem 逻辑
- **选择**: 对于 id: 113，检查获取次数而不是是否拥有
- **原因**: 允许同一个法宝重复掉落
- **实现**: 
  - 统计玩家拥有 id: 113 的次数
  - 如果次数 < 4，则可以掉落
  - 其他法宝保持原有逻辑（只能获得一次）

### 决策3：移除升级选项
- **选择**: 完全移除 id: 9（多重射击）升级
- **原因**: 避免重复，多重射击只能通过法宝获得
- **影响**: 需要清理 game.js 中的相关逻辑

## Risks / Trade-offs

- **风险**: 法宝掉落太随机，玩家可能一直拿不到
  - **缓解**: Boss掉落3个选项，概率较高

- **风险**: 4次上限可能太低
  - **缓解**: 后期可以根据反馈调整上限

- **权衡**: 失去了"稳定获得多重射击"的途径
  - **接受**: 随机性是游戏乐趣的一部分
