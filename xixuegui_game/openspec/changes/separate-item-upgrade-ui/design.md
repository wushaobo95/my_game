## Context

当前代码结构：
- `#upgradeScreen` 元素使用 CSS `::before` 伪元素显示硬编码的"选择技能"标题
- `showUpgradeScreen()` 和 `showItemChoiceScreen()` 都使用相同的卡片样式 `.upgrade-card`
- 技能和法宝选择UI在视觉上是完全一样的

## Goals / Non-Goals

**Goals:**
- 创建视觉上可区分的技能选择和法宝选择界面
- 技能选择：蓝绿色调，代表成长和能力提升
- 法宝选择：金橙色调，代表稀有宝物
- 通过JavaScript动态控制标题和样式
- 保持向后兼容，不破坏现有功能

**Non-Goals:**
- 不改变游戏机制或平衡性
- 不修改状态面板(`#statusPanel`)中技能和法宝的显示方式
- 不添加新的选择逻辑或选项数量

## Decisions

### 1. 配色方案选择
**选择**: 
- 技能选择：蓝绿色系 (`#22aa66` / `#44cc88`) - 代表自然、成长
- 法宝选择：金橙色系 (`#cc8822` / `#ffaa44`) - 代表稀有、珍贵

**理由**: 游戏主题是卡皮巴拉自然风，绿色系与技能(成长)更契合；金色系天然代表珍贵道具。

### 2. 标题实现方式
**选择**: 在HTML中添加显式标题元素 `<div id="upgradeTitle"></div>`，通过JavaScript动态设置文本

**替代方案**: 使用CSS类切换伪元素内容
**选择理由**: 伪元素无法通过JavaScript动态修改内容，显式元素更灵活

### 3. 样式类设计
**选择**: 
- 保留 `.upgrade-card` 作为基础样式
- 添加 `.skill-selection` 和 `.item-selection` 作为容器类
- 通过容器类控制子元素样式，而非为每个卡片单独添加类

**理由**: 减少DOM操作，通过CSS嵌套规则实现样式切换

### 4. 边框和视觉效果
**选择**:
- 技能：绿色虚线边框，绿色发光阴影
- 法宝：金色实线边框，金色发光阴影，更强的发光效果

**理由**: 金色系天然更醒目，符合法宝稀有的设定

## Risks / Trade-offs

- **[Risk]** 新CSS类可能与现有样式冲突 → 使用高特异性选择器 (`#upgradeScreen.skill-selection`)
- **[Risk]** 颜色对比度不足影响可读性 → 选用WCAG AA兼容的颜色组合
- **[Trade-off]** 法宝选择UI更醒目可能让玩家觉得技能选择不重要 → 通过文案和教程引导玩家理解两者重要性

## Migration Plan

1. 更新HTML结构，添加标题元素
2. 添加新CSS样式，保留旧样式作为后备
3. 更新JavaScript逻辑，添加样式类切换
4. 测试两种选择界面是否正常显示
5. 移除不再使用的旧CSS伪元素样式（可选）
