## Context

当前游戏有两种掉落系统：
1. 永久法宝（ITEMS）：Boss死亡时掉落，玩家拾取后永久生效
2. 临时Buff道具（BUFF_ITEMS）：普通敌人死亡时有5%概率掉落，立即激活临时效果

本次重构针对永久法宝系统。当前实现中，Boss死亡时调用`trySpawnItem()`从可用法宝池中随机选择一个，创建`ItemPickup`对象。玩家拾取后直接应用法宝效果。

## Goals / Non-Goals

**Goals:**
- 将Boss死亡掉落改为统一法宝物品
- 拾取时显示选择界面，从3个法宝中选择一个（不放回抽取）
- 法宝池耗尽时不再掉落法宝
- 保持现有法宝效果和平衡性

**Non-Goals:**
- 不改变临时Buff道具系统
- 不增加新的法宝类型
- 不改变法宝效果和数值

## Decisions

### 1. 统一法宝物品的实现方式
**决策**: 修改现有`ItemPickup`类，添加`isUnifiedItem`标志
**理由**: 
- 最小化代码改动
- 复用现有绘制和碰撞检测逻辑
- 避免创建新类带来的维护负担
**替代方案**: 创建新的`UnifiedItemPickup`类。缺点：代码重复。

### 2. 选择界面的实现
**决策**: 复用现有升级选择界面（`upgradeScreen`）
**理由**:
- 界面风格一致
- 减少HTML/CSS改动
- 玩家已熟悉操作方式
**替代方案**: 创建新的选择界面。缺点：增加维护成本。

### 3. 法宝池管理
**决策**: 继续使用现有`ITEMS`数组和`trySpawnItem()`逻辑
**理由**:
- 现有逻辑已经实现了不放回抽取（过滤已拥有的法宝）
- 只需修改返回值，不再返回具体法宝对象
**替代方案**: 创建新的法宝池管理器。缺点：过度设计。

### 4. 选择界面触发时机
**决策**: 拾取时立即显示选择界面，暂停游戏
**理由**:
- 与升级选择体验一致
- 避免玩家在战斗中分心
**替代方案**: 积攒后显示。缺点：增加UI复杂度。

## Risks / Trade-offs

### 风险1: 法宝池耗尽后Boss不再掉落
**影响**: 游戏后期Boss击杀奖励减少
**缓解**: 在代码中添加注释，说明后续可扩展为掉落其他奖励（如经验、临时Buff等）

### 风险2: 选择界面与升级界面混淆
**影响**: 玩家可能分不清是升级还是法宝选择
**缓解**: 修改界面标题为"选择法宝"，使用不同的视觉风格（如颜色）

### 风险3: 法宝池管理复杂度
**影响**: 需要正确维护已拥有法宝列表
**缓解**: 使用现有`player.acquiredUpgrades`数组，确保逻辑正确

## Migration Plan
